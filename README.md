# Todo App Frontend (учебный CRUD-шаблон)

Простой фронтенд-проект для обучения REST API на примере задач.

Проект повторяет учебный паттерн из `calculator-frontend`:
- `GET` список сущностей
- `POST` создание
- `PATCH` редактирование
- `DELETE` удаление

## Запуск

```bash
npm install
npm start
```

Приложение будет доступно на `http://localhost:3000`.

## Ожидаемое API

Базовый URL: `http://localhost:8080`

### 1) Получить список задач (`GET /tasks`)

Тело запроса: **отсутствует**.

Ожидаемый ответ:
- Статус: `200 OK`
- Тело ответа: JSON-массив задач

```json
[
  {
    "id": 1,
    "title": "Сделать ДЗ",
    "completed": false
  },
  {
    "id": 2,
    "title": "Проверить PR",
    "completed": true
  }
]
```

Если задач нет:

```json
[]
```

### 2) Создать задачу (`POST /tasks`)

Тело запроса:

```json
{
  "title": "Подготовить урок"
}
```

Ожидаемый ответ:
- Статус: `201 Created` (допустимо `200 OK`)
- Тело ответа: созданная задача

```json
{
  "id": 3,
  "title": "Подготовить урок",
  "completed": false
}
```

### 3) Обновить задачу (`PATCH /tasks/{id}`)

Тело запроса:

```json
{
  "title": "Подготовить урок по REST",
  "completed": true
}
```

Ожидаемый ответ:
- Статус: `200 OK`
- Тело ответа: обновленная задача

```json
{
  "id": 3,
  "title": "Подготовить урок по REST",
  "completed": true
}
```

### 4) Удалить задачу (`DELETE /tasks/{id}`)

Тело запроса: **отсутствует**.

Ожидаемый ответ:
- Статус: `204 No Content`
- Тело ответа: **отсутствует**

## Что можно объяснять ученикам

- связь UI-элементов и HTTP-методов
- хранение состояния формы и списка в React (`useState`)
- загрузка данных при старте (`useEffect`)
- обработка ошибок API
- перезагрузка списка после изменения данных
